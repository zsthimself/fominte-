# pSEO 工作流程

## 概述

pSEO（程序化 SEO）通过批量生成"产品×应用场景"的着陆页，覆盖大量长尾关键词，提升自然搜索流量。

---

## 执行步骤

### 1. 场景研究

**目的**：确定目标产品的应用场景和相关关键词

**操作**：
1. 分析产品的主要用途和目标客户
2. 使用 SEO 工具调研相关关键词
3. 将场景数据整理为 JSON 格式

**产出**：`data/application-scenarios.json`

---

### 2. 生成矩阵

**目的**：创建产品×场景的完整矩阵

**操作**：
```bash
cd scripts/pseo-content-generator
node generate-matrix.js
```

**产出**：`data/product-scenario-matrix.json`

---

### 3. 应用模板

**目的**：为每个页面生成 SEO 优化的内容

**操作**：
1. 根据模板生成标题、描述、正文
2. 应用 Formal Wear 等特定模板

**产出**：更新后的矩阵数据

---

### 4. 导出 CSV

**目的**：生成 Shopline 可导入的 CSV 文件

**操作**：
```bash
node export-new-products.js
```

**产出**：`data/pseo-new-products.csv`

---

### 5. 导入 Shopline

**目的**：将生成的产品页面导入 Shopline

**操作**：
1. 登录 Shopline 后台
2. 产品 → 导入
3. 上传 CSV 文件

---

### 6. 提交索引

**目的**：加速页面被搜索引擎收录

**操作**：
1. 生成 HTML 站点地图
2. 提交到 Google Search Console
3. （可选）使用 Indexing API 批量提交

**工具**：`scripts/` 下的索引相关脚本

---

## 注意事项

- 确保每个页面内容独特，避免重复
- 监控索引状态，及时处理问题页面
- 定期更新场景数据
